Inventory Management System Requirements

I am building a multi-company inventory management SaaS for footwear shops.

The first company using it will be Step Enterprise.

The system must be mobile-friendly, installable as a PWA, and optimized for daily inventory operations.

Naming Conventions
Prisma

Use:

PascalCase -> Models
camelCase -> Fields

Example:

model Product {
  articleNo String
  articleName String
  createdAt DateTime
}
PostgreSQL

Use:

snake_case

Example:

products
product_variants

article_no
article_name
created_at

Use:

@map()
@@map()
IDs

Keep:

id String @id @default(cuid())

Do NOT use:

Int @id @default(autoincrement())
Multi-Company

Every business record belongs to a company through:

companyId String

Relation:

company Company @relation(
  fields: [companyId],
  references: [id]
)

Keep using Company.id as the foreign key.

Do NOT use company code as the foreign key.

Company code is only for display purposes.

Main Goal

Build a production-ready inventory management system that:

Supports multiple companies
Tracks stock accurately
Tracks purchases and partial deliveries
Tracks sales
Tracks stock transfers
Maintains inventory history
Supports role-based access

Can later be sold as a SaaS product
